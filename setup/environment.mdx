## Environment File

Ecom uses Laravel's environment configuration system. All environment-specific settings are stored in the `.env` file at the root of your application.

<Info>
The `.env` file is automatically created from `.env.example` during installation. Never commit `.env` to version control.
</Info>

## Creating Environment File

<Steps>
  <Step title="Copy Example File">
    ```bash
    cp .env.example .env
    ```
  </Step>
  
  <Step title="Generate Application Key">
    ```bash
    php artisan key:generate
    ```
  </Step>
  
  <Step title="Configure Environment Variables">
    Edit `.env` file with your specific settings
  </Step>
</Steps>

## Core Environment Variables

### Application Settings

```bash .env
# Application Identity
APP_NAME=Laravel
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost
```

<AccordionGroup>
  <Accordion title="APP_NAME">
    **Type:** String  
    **Default:** `Laravel`  
    **Description:** The name of your application used in emails and notifications
    
    ```bash
    APP_NAME="My Ecom Store"
    ```
  </Accordion>

  <Accordion title="APP_ENV">
    **Type:** String  
    **Default:** `local`  
    **Options:** `local`, `staging`, `production`  
    **Description:** Current environment mode
    
    ```bash
    # Development
    APP_ENV=local
    
    # Production
    APP_ENV=production
    ```
  </Accordion>

  <Accordion title="APP_KEY">
    **Type:** String  
    **Default:** Empty (generated)  
    **Description:** Encryption key for the application
    
    <Warning>
    This is automatically generated by `php artisan key:generate`. Never share this key.
    </Warning>
    
    ```bash
    APP_KEY=base64:randomgeneratedkey...
    ```
  </Accordion>

  <Accordion title="APP_DEBUG">
    **Type:** Boolean  
    **Default:** `true` (local), `false` (production)  
    **Description:** Enable detailed error messages
    
    <Warning>
    **CRITICAL:** Always set to `false` in production to prevent exposing sensitive information.
    </Warning>
    
    ```bash
    # Development
    APP_DEBUG=true
    
    # Production
    APP_DEBUG=false
    ```
  </Accordion>

  <Accordion title="APP_URL">
    **Type:** URL  
    **Default:** `http://localhost`  
    **Description:** Base URL of your application
    
    ```bash
    # Local
    APP_URL=http://localhost:8000
    
    # Production
    APP_URL=https://yourdomain.com
    ```
  </Accordion>
</AccordionGroup>

### Logging Configuration

```bash .env
LOG_CHANNEL=stack
```

**Available Channels:**
- `stack` - Multiple log channels (default)
- `single` - Single log file
- `daily` - Daily log files
- `slack` - Slack notifications
- `syslog` - System log
- `errorlog` - PHP error log

## Database Configuration

```bash .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=homestead
DB_USERNAME=homestead
DB_PASSWORD=secret
```

<AccordionGroup>
  <Accordion title="MySQL Configuration">
    **Default Port:** 3306  
    **Configured in:** `config/database.php:42`
    
    ```bash .env
    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=ecom_database
    DB_USERNAME=ecom_user
    DB_PASSWORD=secure_password
    ```
    
    <Note>
    MySQL uses `utf8mb4` charset and `utf8mb4_unicode_ci` collation by default.
    </Note>
  </Accordion>

  <Accordion title="PostgreSQL Configuration">
    **Default Port:** 5432  
    **Configured in:** `config/database.php:57`
    
    ```bash .env
    DB_CONNECTION=pgsql
    DB_HOST=127.0.0.1
    DB_PORT=5432
    DB_DATABASE=ecom_database
    DB_USERNAME=postgres
    DB_PASSWORD=secure_password
    ```
  </Accordion>

  <Accordion title="SQLite Configuration">
    **Configured in:** `config/database.php:36`
    
    ```bash .env
    DB_CONNECTION=sqlite
    DB_DATABASE=/absolute/path/to/database.sqlite
    ```
  </Accordion>

  <Accordion title="SQL Server Configuration">
    **Default Port:** 1433  
    **Configured in:** `config/database.php:70`
    
    ```bash .env
    DB_CONNECTION=sqlsrv
    DB_HOST=localhost
    DB_PORT=1433
    DB_DATABASE=ecom_database
    DB_USERNAME=sa
    DB_PASSWORD=secure_password
    ```
  </Accordion>
</AccordionGroup>

## Cache & Session Configuration

```bash .env
CACHE_DRIVER=file
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120
```

<AccordionGroup>
  <Accordion title="CACHE_DRIVER">
    **Default:** `file`  
    **Options:** `file`, `redis`, `memcached`, `database`, `array`
    
    ```bash
    # Development
    CACHE_DRIVER=file
    
    # Production (recommended)
    CACHE_DRIVER=redis
    ```
  </Accordion>

  <Accordion title="QUEUE_CONNECTION">
    **Default:** `sync`  
    **Options:** `sync`, `database`, `redis`, `beanstalkd`, `sqs`
    
    ```bash
    # Development
    QUEUE_CONNECTION=sync
    
    # Production
    QUEUE_CONNECTION=redis
    ```
    
    <Note>
    When using `redis` or `database`, you need to run queue workers:
    ```bash
    php artisan queue:work
    ```
    </Note>
  </Accordion>

  <Accordion title="SESSION_DRIVER & SESSION_LIFETIME">
    **SESSION_DRIVER Default:** `file`  
    **SESSION_LIFETIME Default:** `120` (minutes)
    
    ```bash
    SESSION_DRIVER=file
    SESSION_LIFETIME=120
    
    # For production with Redis
    SESSION_DRIVER=redis
    SESSION_LIFETIME=1440
    ```
  </Accordion>
</AccordionGroup>

## Redis Configuration

```bash .env
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
```

Redis is configured in `config/database.php:107` with two databases:

<CodeGroup>
```bash Default Connection
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
REDIS_DB=0
```

```bash Cache Connection
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
REDIS_CACHE_DB=1
```
</CodeGroup>

<Info>
Ecom uses `predis` as the Redis client (`config/database.php:109`).
</Info>

## Mail Configuration

```bash .env
MAIL_DRIVER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
```

<AccordionGroup>
  <Accordion title="SMTP Configuration">
    ```bash .env
    MAIL_DRIVER=smtp
    MAIL_HOST=smtp.gmail.com
    MAIL_PORT=587
    MAIL_USERNAME=your-email@gmail.com
    MAIL_PASSWORD=your-app-password
    MAIL_ENCRYPTION=tls
    MAIL_FROM_ADDRESS=noreply@yourdomain.com
    MAIL_FROM_NAME="${APP_NAME}"
    ```
  </Accordion>

  <Accordion title="Mailtrap (Testing)">
    ```bash .env
    MAIL_DRIVER=smtp
    MAIL_HOST=smtp.mailtrap.io
    MAIL_PORT=2525
    MAIL_USERNAME=your-mailtrap-username
    MAIL_PASSWORD=your-mailtrap-password
    MAIL_ENCRYPTION=null
    ```
  </Accordion>

  <Accordion title="Mailgun">
    ```bash .env

    MAILGUN_DOMAIN=your-domain.com
    ```

## Security Best Practices

<Warning>
**Critical Security Rules:**

1. Never commit `.env` to version control
2. Always set `APP_DEBUG=false` in production
3. Use strong, unique `APP_KEY` for each environment
4. Use environment variables for all sensitive data
5. Restrict file permissions: `chmod 600 .env`
6. Regularly rotate credentials and keys
</Warning>

## Verifying Configuration

Test your environment configuration:

```bash
# Check current environment
php artisan env

# Test database connection
php artisan migrate:status

# Clear and cache configuration
php artisan config:clear
php artisan config:cache
```
