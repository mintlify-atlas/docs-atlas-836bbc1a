## Get Profile Counters

<api method="GET" endpoint="/api/v2/profile/counters" auth="sanctum">
  Get user's cart, wishlist, and order counts
</api>

### Response

```json
{
  "cart_item_count": 3,
  "wishlist_item_count": 5,
  "order_count": 12
}
```

### Example

```bash
curl -X GET https://your-domain.com/api/v2/profile/counters \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Update Profile

<api method="POST" endpoint="/api/v2/profile/update" auth="sanctum">
  Update user profile information
</api>

### Request

<ParamField body="name" type="string">
  User's full name
</ParamField>

<ParamField body="phone" type="string">
  Phone number
</ParamField>

<ParamField body="password" type="string">
  New password (if changing password)
</ParamField>

### Response

```json
{
  "result": true,
  "message": "Profile information updated"
}
```

### Example

```bash
curl -X POST https://your-domain.com/api/v2/profile/update \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "phone": "+1234567890"
  }'
```

## Update Profile Image

<api method="POST" endpoint="/api/v2/profile/update-image" auth="sanctum">
  Update user profile picture
</api>

### Request

<ParamField body="image" type="string" required>
  Base64 encoded image
</ParamField>

<ParamField body="filename" type="string" required>
  Image filename with extension
</ParamField>

### Response

<ResponseField name="result" type="boolean">
  Success status
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="path" type="string">
  URL to uploaded image
</ResponseField>

### Example

```bash
curl -X POST https://your-domain.com/api/v2/profile/update-image \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "image": "base64_encoded_image_data",
    "filename": "profile.jpg"
  }'
```

<Note>
Supported image formats: jpg, jpeg, png, svg, webp, gif
</Note>
