## Overview

The Ecom API uses Laravel Sanctum for authentication. After successful login or signup, you'll receive an access token that must be included in subsequent API requests.

## Authentication Flow

1. User signs up or logs in
2. API returns an access token
3. Include token in `Authorization` header for protected endpoints

## Register User

<api method="POST" endpoint="/api/v2/auth/signup" auth="none">
  Register a new user account
</api>

### Request

<ParamField body="name" type="string" required>
  User's full name
</ParamField>

<ParamField body="email_or_phone" type="string" required>
  Email address or phone number (based on register_by)
</ParamField>

<ParamField body="register_by" type="string" required>
  Registration method: `email` or `phone`
</ParamField>

<ParamField body="password" type="string" required>
  Password (minimum 6 characters)
</ParamField>

<ParamField body="password_confirmation" type="string" required>
  Password confirmation (must match password)
</ParamField>

<ParamField body="g-recaptcha-response" type="string">
  reCAPTCHA response (if enabled)
</ParamField>

### Response

<ResponseField name="result" type="boolean">
  Success status
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="access_token" type="string">
  Bearer token for authentication
</ResponseField>

<ResponseField name="token_type" type="string">
  Token type (always "Bearer")
</ResponseField>

<ResponseField name="user" type="object">
  User information
  <Expandable>
    <ResponseField name="id" type="integer">
      User ID
    </ResponseField>
    <ResponseField name="type" type="string">
      User type (customer, seller, delivery_boy)
    </ResponseField>
    <ResponseField name="name" type="string">
      User's name
    </ResponseField>
    <ResponseField name="email" type="string">
      User's email
    </ResponseField>
    <ResponseField name="phone" type="string">
      User's phone number
    </ResponseField>
    <ResponseField name="avatar_original" type="string">
      Profile image URL
    </ResponseField>
    <ResponseField name="email_verified" type="boolean">
      Email verification status
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -X POST https://your-domain.com/api/v2/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email_or_phone": "john@example.com",
    "register_by": "email",
    "password": "password123",
    "password_confirmation": "password123"
  }'
```

## Login

<api method="POST" endpoint="/api/v2/auth/login" auth="none">
  Authenticate user and receive access token
</api>

### Request

<ParamField body="email" type="string" required>
  Email address or phone number
</ParamField>

<ParamField body="password" type="string" required>
  User password
</ParamField>

<ParamField body="login_by" type="string" required>
  Login method: `email` or `phone`
</ParamField>

<ParamField body="user_type" type="string">
  User type: `customer`, `seller`, or `delivery_boy`
</ParamField>

<ParamField body="identity_matrix" type="string">
  Identity verification matrix
</ParamField>

### Response

Same structure as signup response.

### Example

```bash
curl -X POST https://your-domain.com/api/v2/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "password123",
    "login_by": "email"
  }'
```

## Get User Info

<api method="GET" endpoint="/api/v2/auth/user" auth="sanctum">

</api>


## Using the Access Token

Include the access token in the `Authorization` header for all protected endpoints:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

Example:

```bash
curl -X GET https://your-domain.com/api/v2/cart-summary \
  -H "Authorization: Bearer 1|abc123def456..."
```
